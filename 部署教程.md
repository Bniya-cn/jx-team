# JX-Team 本地部署教程

这份教程面向零基础同学，目标是把项目在本机完整跑起来。默认先按 **Windows** 写法说明，同时补充 macOS / Linux 的共通注意点。

演示账号：`Guest / password`

---

## 1. 环境要求

| 工具 | 建议版本 |
|------|----------|
| Ruby | `2.5.1` |
| Bundler | `2.2.3` |
| Node.js | `14+`，推荐 `18 LTS` 或 `20 LTS` |
| PostgreSQL | `16` |
| Git | 最新版 |

如果你的电脑已经有别的 Ruby、Node、PostgreSQL 版本，建议先确认 PATH 不会串版本。

---

## 2. 下载代码

```bash
git clone <你的仓库地址> jx-team-main
cd jx-team-main
```

如果老师发的是压缩包，解压后进入项目根目录，确保能看到：

- `Gemfile`
- `package.json`
- `config/`
- `frontend/`

---

## 3. 安装 Ruby 与 Bundler

项目固定使用 `Ruby 2.5.1`，请不要直接用 3.x。

### Windows

1. 打开 RubyInstaller 历史版本下载页
2. 安装 `Ruby+Devkit 2.5.1-1 (x64)`
3. 勾选 `Add Ruby executables to your PATH`
4. 安装完成后重新打开终端

验证：

```bash
ruby -v
gem -v
```

安装 Bundler：

```bash
gem install bundler -v 2.2.3
```

### macOS / Linux

如果系统里已经有兼容的 Ruby 2.5.1，可以直接使用；否则建议用 `rbenv` / `rvm` 安装对应版本。

验证：

```bash
ruby -v
bundle -v
```

---

## 4. 安装 Node.js

安装完成后确认：

```bash
node -v
npm -v
```

项目前端构建会用到 `npm run build`。

---

## 5. 安装 PostgreSQL

项目数据库使用 PostgreSQL。

### Windows

1. 安装 PostgreSQL 16
2. 记住 `postgres` 超级用户密码
3. 保持默认端口 `5432`
4. 安装后确认数据库服务正在运行

### macOS / Linux

可使用 Homebrew、apt、官方安装包均可，只要本机能正常启动 PostgreSQL 即可。

验证数据库是否可用：

```bash
psql --version
pg_isready
```

---

## 6. 配置数据库连接

当前项目默认数据库名是：

- `jx_team_development`
- `jx_team_test`

`config/database.yml` 已经按 PostgreSQL 配置好。

### Windows 需要注意

Windows 下如果本机 `pg` 连接不通，常见原因是没有启用 TCP 连接。此时在 `development` 和 `test` 中补上：

```yaml
development:
  <<: *default
  database: jx_team_development
  host: localhost
  username: postgres
  password: 你的postgres密码

test:
  <<: *default
  database: jx_team_test
  host: localhost
  username: postgres
  password: 你的postgres密码
```

如果你使用的是别的 PostgreSQL 用户名，也可以改成对应账号，但要保证该账号有建库权限。

---

## 7. 安装后端依赖

在项目根目录执行：

```bash
bundle install
```

常见补充说明：

- 如果提示 `Could not find bundler`，先执行 `gem install bundler -v 2.2.3`
- 如果提示 `mimemagic` 相关错误，按系统要求先设置环境变量再安装
- 如果 `pg` 编译失败，确认 PostgreSQL 的 `bin` 目录已加入 PATH

---

## 8. 初始化数据库

首次部署建议直接：

```bash
bundle exec rails db:setup
```

这一步会：

1. 创建数据库
2. 执行迁移
3. 导入种子数据

如果你需要重建数据，可以执行：

```bash
bundle exec rails db:drop db:create db:migrate db:seed
```

当前种子会把 `src/mock/games.json` 里的 catalog 导入到 Rails 数据库里，所以重新 seed 后首页和详情页的数据会统一回到数据库真源。

---

## 9. 安装并构建前端

```bash
npm install
npm run build
```

说明：

- `npm install` 安装 React、Redux、Webpack 等前端依赖
- `npm run build` 生成前端打包资源
- 如果你在改 `frontend/` 下代码，每次改完后都要重新执行 `npm run build`

如果你希望边改边看效果，可以另开一个终端执行：

```bash
npm run webpack
```

### 9.1 如需刷新游戏 catalog

如果你更新了抓取脚本，或者想重新生成一批游戏数据，可以按下面顺序执行：

```bash
npm run fetch:steam-games
bundle exec rails db:drop db:create db:migrate db:seed
npm run build
```

说明：

- 抓取脚本会更新 `src/mock/games.json`
- 重新 seed 后，数据库里的游戏主数据会同步刷新
- 如果只是改了前端样式，不需要执行这一步

## 10. 启动服务

```bash
bundle exec rails server
```

看到类似下面的输出就说明成功了：

```text
Listening on http://127.0.0.1:3000
```

浏览器打开：

```text
http://localhost:3000
```

停止服务时按 `Ctrl + C`。

## 11. 翻译配置（可选）

如果你想启用 DeepSeek 翻译：

1. 复制 `.env.example` 为 `.env`
2. 填写：

```env
DEEPSEEK_API_KEY=sk-你的密钥
```

3. 重新启动 Rails 服务

如果没有配置 Key：

- 顶栏翻译 UI 仍然可见
- 翻译接口会返回占位译文
- 项目仍然可以正常演示

## 12. 验证清单

部署完成后，建议按下面顺序确认：

- `http://localhost:3000` 可以打开首页
- `Guest / password` 可以登录
- 首页轮播和游戏列表正常显示

## 13. 常见问题

### 13.1 Ruby 版本不对

如果提示 Ruby 版本不是 `2.5.1`，说明当前 PATH 里不是项目要求的 Ruby。请重新打开终端并确认版本。

### 13.2 bundler 不存在

```bash
gem install bundler -v 2.2.3
```

### 13.3 数据库连不上

先确认 PostgreSQL 已启动，再检查 `config/database.yml` 里的 `host`、`username`、`password`。

### 13.4 pg gem 编译失败

先确认 PostgreSQL 安装正常，并把其 `bin` 目录加入 PATH，然后重新执行：

```bash
bundle config build.pg --with-pg-config="C:/Program Files/PostgreSQL/16/bin/pg_config.exe"
bundle install
```

### 13.5 找不到 `mimemagic`

先按系统要求设置相应环境变量，再执行 `bundle install`。

### 13.6 页面可以开，但内容不对

先执行：

```bash
bundle exec rails db:drop db:create db:migrate db:seed
npm run build
```

如果仍有问题，再检查浏览器控制台和 Rails 终端报错。
