# 交接计划与未完成事项

这份文档用于把当前项目的已完成内容、未完成内容和下一步执行顺序收束到一起，方便队友继续接手。

---

## 1. 当前已完成

- Rails 主链路已经接通
- 游戏 catalog 已导入数据库
- 前端商品读取路径已经统一到 API
- 用户评测、购买、库存、愿望单、推荐、动态等功能都能串起来
- 后台管理骨架已经有了
- 多语言和翻译 UI 已经接上
- 部署教程、技术栈、功能说明、演示说明已经补齐

---

## 2. 最优先需要继续做的事

### 2.1 修首页布局

当前首页还有两个明显问题：

- 横向溢出
- 右侧空白过大，左侧内容被挤压

建议优先检查这些文件：

- `app/assets/stylesheets/base/main.scss`
- `app/assets/stylesheets/components/sidebar.scss`
- `app/assets/stylesheets/components/featured_carousel.scss`

目标不是简单加 `overflow-x: hidden`，而是把布局结构本身收紧。

### 2.2 补全详情页内容

用户反馈详情页缺少：

- 用户测评
- 推荐配置 / 最低配置
- 更完整的推荐模块

建议优先补这些文件：

- `frontend/components/games/show/game_detail.jsx`
- `frontend/components/games/show/buy_now_bar.jsx`
- `frontend/components/reviews/reviews_index.jsx`
- `frontend/components/reviews/new_review_form.jsx`

### 2.3 继续完善中文化

当前中文模式下仍可能出现：

- 英文游戏名
- 英文简介
- 英文分类
- 英文链接或按钮

建议先做统一 helper，再逐个页面替换，而不是零散修补。

### 2.4 融入校徽

用户已经提供了江西师范大学校徽文件，建议把它真正纳入：

- 顶部品牌位
- 左侧边栏品牌位
- favicon 或启动标识

这样项目会从“Steam 风格仿站”更自然地转成“校园项目作品”。

### 2.5 清洗 Steam 抓取脚本

当前抓取脚本虽然扩了 catalog，但还存在误命中风险，需要继续修：

- `scripts/fetch-steam-games.js`
- `src/mock/games.json`

重点是：

- 排除 DLC、Soundtrack、Bundle 等误命中
- 加强 appid 白名单和置信度判断
- 避免把错误条目重新 seed 回数据库

---

## 3. 建议的执行顺序

1. 先修首页布局和横向溢出
2. 再补详情页的评测和推荐内容
3. 同步推进中文化统一
4. 把校徽接进品牌区
5. 最后清洗抓取脚本和 catalog 数据

## 5. 验证建议

接手后建议按这个顺序验证：

1. `bundle exec rails test`
2. `npm run test:locale`
3. `npm run build`
4. 启动 Rails 后打开首页
5. 检查是否还有横向溢出
6. 打开详情页看评测区和推荐区是否补齐
7. 走一遍 `详情 -> 购物车 -> 购买 -> 库存 -> 评测`

---

## 6. 交接结论

现在项目已经具备交接条件，后续工作的核心不再是“能不能跑”，而是：

- 页面是否更稳
- 文案是否更完整
- 品牌是否更统一
- 演示内容是否更像一套完整作品

只要把上面几项继续补完，这个项目就可以比较自然地交给队友继续深化。
